# RULES YOU MUST FOLLOW

## Processus de développement

- **Always ask for a "GO" before writing or making changes in the code**
  - Toujours demander une confirmation avant d'implémenter des modifications

- **Think twice and explain your reasoning before asking for the "GO"**
  - Réfléchir et expliquer le raisonnement avant de demander la validation

## Principes de conception

- **Think and propose the simplest solution**
  - Appliquer les principes "Keep It Simple Stupid" (KISS) et "YAGNI" (You Aren't Gonna Need It)

- **Propose 3 options for each action**
  - Toujours présenter trois options différentes pour chaque action envisagée

- **Make sure to structure your code in a modular way**
  - Structurer le code de manière modulaire pour faciliter la maintenance

## Qualité du code

- **Add all the detailed console logs to facilitate the debugging**
  - Ajouter des logs détaillés dans la console pour faciliter le débogage

- **Put comments in your code to facilitate debugging and reusability**
  - Commenter le code pour améliorer sa lisibilité et sa réutilisabilité

## Gestion des tâches

- **Breaks down complex tasks into a workflow of smaller, more manageable tasks**
  - Décomposer les tâches complexes en sous-tâches plus simples

- **Before any code generation or modification: always put your plan in the todo list**
  - Créer systématiquement un plan dans la todo list avant toute modification

- **Keep your to-do list up to date with your tasks**
  - Maintenir la todo list à jour en permanence
